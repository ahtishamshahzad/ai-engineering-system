# Prompt: Feature Plan

> Tool-neutral starter. Plan a feature within existing architecture.

---

Plan a **feature** (`.ai/` canonical; `.ai/workflows/new-feature.md`).

**Feature:** <what capability, for whom>

Do this:
1. Plan within the existing architecture, no scope creep (`.ai/skills/feature-planning`) → `.ai/templates/FEATURE.md`.
2. Break it into tasks with acceptance criteria → `.ai/templates/TASK.md`.
3. Define the test approach and required cases (`.ai/skills/testing/testing-selection`) → `.ai/templates/TEST_PLAN.md`.
4. Note contracts/data touched; if it needs new apps/stack or an architecture change, **stop and escalate** (wrong workflow).

Keep it scoped. Present the plan for Gate 4 approval before implementing.
