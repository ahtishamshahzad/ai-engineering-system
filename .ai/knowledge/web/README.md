# Knowledge — Web

> Durable, reusable web/frontend engineering knowledge. Read only when the task needs it. **Not a tutorial dump.** Every entry carries a trust label; version-sensitive entries carry the metadata block — see `../README.md`. Starts empty.

## What belongs here

- Rendering-strategy trade-offs (SSR/SSG/CSR/edge) and when each fits.
- Accessibility patterns and browser-behavior gotchas.
- Framework constraints and version-specific pitfalls (with versions).

## What does NOT belong here

- Generic framework tutorials — link the docs.
- Project-specific page/dashboard notes → `../../references/web/`.
- Canonical rules → `../../system/`.

## Quality & source requirements

- Reusable, non-obvious, with limits and the *why*; cite the source.
- Record `affected_versions` for framework-specific notes; undated version claims are **unverified**.
- Label trust: verified-current / project-convention / opinion / deprecated / unverified.

## Update & deprecation

- Re-verify against current framework/browser versions; mark superseded guidance **deprecated** with a successor.

## How skills reference this

- Web skills and `../../skills/security/web-security` link here for durable/version-sensitive knowledge rather than embedding it.
