# Knowledge — Testing

> Durable, reusable testing knowledge. Read only when the task needs it. **Not a tutorial dump.** Every entry carries a trust label; tool-version-sensitive entries carry the metadata block — see `../README.md`. Starts empty.

## What belongs here

- Reusable testing-strategy lessons (what to test at which level, by risk).
- Tool behaviors and pitfalls (Playwright/Maestro/Supertest/Jest/Vitest), with versions.
- Recurring flakiness patterns and their root-cause fixes.

## What does NOT belong here

- Generic testing tutorials — link the docs.
- Project-specific test patterns → `../../references/testing/`.
- Canonical rules → `../../system/` (`TESTING_SELECTION_RULES.md`).

## Quality & source requirements

- Reusable, non-obvious; cite the source (tool docs, flaky-test investigation).
- Record `affected_versions` for tool-specific notes; undated tool claims are **unverified**.
- Label trust: verified-current / project-convention / opinion / deprecated / unverified.

## Update & deprecation

- Re-verify against current tool versions; mark superseded workarounds **deprecated** when the tool fixes them.
