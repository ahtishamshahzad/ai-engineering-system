# Knowledge — Mobile

## What belongs here

- Expo vs RN CLI constraints and platform gotchas learned in practice.
- iOS/Android behavior differences worth remembering; store-policy constraints.
- Native-module/config-plugin lessons; version-specific pitfalls (with versions).

## What does NOT belong here

- Generic React Native tutorials — link the docs.
- Project-specific screen/nav notes → `../../references/mobile/`.
- Canonical rules → `../../system/`.

## Quality & source requirements

- Reusable and non-obvious; cite the source (official docs, release notes, incident).
- **Record `affected_versions`** — a mobile note without versions is likely stale fast; treat undated version-specific notes as **unverified**.
- Label trust: verified-current / project-convention / opinion / deprecated / unverified.

## Update & deprecation

- Re-verify against current Expo/RN versions; mark superseded workarounds **deprecated** when the platform fixes them.

## How skills reference this
