# References — Web / SEO

> Topic reference folder. Agents read this folder **only when the active task needs it** (`../../../system/CONTEXT_MANAGEMENT_RULES.md`, `TOKEN_OPTIMIZATION_RULES.md`) — never the whole tree. Starts empty; material is added as the project develops. **Do not create fake reference content.**

## What belongs here

SEO reference material: metadata/title conventions, structured-data (JSON-LD) examples, sitemap/robots notes, canonical-URL and Open Graph patterns, and Core Web Vitals notes.

## Supported file types

Markdown (`.md`) for notes; small snippets in fenced blocks.

## Naming conventions

`kebab-case` by concern (`metadata.md`, `structured-data.md`, `sitemap.md`).

## How agents should use it

Read when a page is public and SEO matters. Apply conventions to public pages; SEO is not relevant to authenticated app screens.

## References are not requirements

Material here is **context, not commitment.** An SEO reference does **not** automatically become a requirement. Requirements/tasks and gate approval govern the build (`../../../system/QUALITY_GATES.md`). Cite when it informs the work; never let it silently drive scope.
