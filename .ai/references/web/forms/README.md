# References — Web / Forms

> Topic reference folder. Agents read this folder **only when the active task needs it** (`../../../system/CONTEXT_MANAGEMENT_RULES.md`, `TOKEN_OPTIMIZATION_RULES.md`) — never the whole tree. Starts empty; material is added as the project develops. **Do not create fake reference content.**

## What belongs here

Web form reference material: form-state and validation patterns, submission/error-UX, accessible form markup, and multi-step form structures.

## Supported file types

Markdown (`.md`) for patterns/notes; small code snippets in fenced blocks.

## Naming conventions

`kebab-case` by form/concern (`checkout-form.md`, `validation-patterns.md`).

## How agents should use it

Read when building web forms. Client validation is UX; the **server enforces**. Ensure accessible labels/errors.

## References are not requirements

Material here is **context, not commitment.** A form reference does **not** automatically become a requirement. Requirements/tasks and gate approval govern the build (`../../../system/QUALITY_GATES.md`). Cite when it informs the work; never let it silently drive scope.
