# References — Web (Topic Index)

> Web reference material, organized by sub-topic. Agents read **only the relevant sub-folder** for the active task (`../../system/CONTEXT_MANAGEMENT_RULES.md`, `TOKEN_OPTIMIZATION_RULES.md`) — never the whole tree. Each sub-folder starts empty; material is added as the project develops. **Do not create fake reference content.**

## Sub-topics

| Folder | Covers |
|--------|--------|
| [`pages/`](pages/README.md) | Page/route layout patterns, rendering strategy, state handling. |
| [`dashboard/`](dashboard/README.md) | Admin/dashboard tables, bulk actions, role-gated UI, reporting. |
| [`forms/`](forms/README.md) | Form state, validation, accessible form UX. |
| [`seo/`](seo/README.md) | Metadata, structured data, sitemap/robots, Core Web Vitals. |

## Conventions

Each terminal sub-folder's README states what belongs there, file types, naming, how agents use it, and that **references do not automatically become requirements**. Server enforces access; client checks are UX. Security lens: `../../skills/security/web-security`.
